## 一、项目定位

**矩阵成本核算引擎**是一个基于线性代数（Leontief 投入产出模型）的企业成本核算系统。核心理念是：**将生产过程中的物料流转和工单流转抽象为稀疏矩阵网络，通过求解逆矩阵一次性算出所有节点（物料+工单）的料、工、费成本**。

与传统逐步结转法不同，本系统采用**平行结转法**（矩阵法），天然支持：
- 成本还原（追溯每笔原始成本去了哪里）
- 跨层级成本穿透（原材料→半成品→成品，自动追踪）
- 大规模并发计算（利用稀疏矩阵，支持万级节点）

**项目路径**: `d:\桌面\公众号\成本核算系统\`
**当前版本**: 3.0（commit: `9d8d214`）
**下一版本**: 4.0（计划中，见 `doc/PLAN_4.0.md`）

## 二、核心理论框架

### 2.1 基础公式

$$X = (I - WD)^{-1} \times F$$

各符号含义：

| 符号 | 含义 | 维度 |
|------|------|------|
| **W** | 物料-工单流转稀疏矩阵 | N×N（N = 物料数 + 工单数）|
| **D** | 完工率阀门对角矩阵 | N×N |
| **F** | 外部投入矩阵（期初/采购/人工/制费） | N×3（料、工、费三列）|
| **X** | 各节点的总成本 | N×3 |

### 2.2 核心思想：物料与工单的"二象性"

这是 3.0 版本最根本的理论突破。传统模型认为：

```
原材料A → 半成品B → 成品C
```

即"物料被物料消耗"。但 3.0 的模型反过来：

```
原材料A → MO1（工单）→ 半成品B → MO2（工单）→ 成品C
```

**核心洞察**：
- 物料不是被物料消耗的，**物料是被工单消耗的**
- 物料也不是从物料产出的，**物料是从工单产出的**
- 因此，**物料和工单构成一个统一的二部图网络**，W 矩阵的行和列同时包含物料和工单

这带来两个关键概念：

| 概念 | 含义 | W矩阵中的体现 |
|------|------|-------------|
| **工单→物料消耗关系** | 物料领用数量 / 可供发出数量 | W[工单行, 物料列] = 消耗比例 |
| **工单→物料产出关系** | 只要工单产出该物料，比例就是100%（不考虑其他工单也产出同物料） | W[物料行, 工单列] = 1.0（或按完工数量分摊） |

### 2.3 W 矩阵的双区结构

W 矩阵（N×N）包含四个区域：

```
           物料列        工单列
物料行  [   0      ]  [ 产出比例  ]   ← 物料←工单（产出）
工单行  [ 消耗比例  ]  [    0     ]   ← 工单←物料（消耗）
```

- **左上角**（物料行×物料列）：始终为 0（物料不直接消耗物料）
- **右上角**（物料行×工单列）：工单→产品的产出比例（通常为 1.0）
- **左下角**（工单行×物料列）：物料→工单的消耗比例（领用/可供发出）
- **右下角**（工单行×工单列）：始终为 0（工单不直接消耗工单）

### 2.4 阀门矩阵 D

D 是一个对角矩阵，对角线上的值 = **完工数量 / (完工数量 + 在产品数量)**。

**材料路径**（D_mat）：
- 物料节点：D = 1.0（物料不存在在产品，全额流转）
- 工单节点：D = 完工率（如 100/(100+25) = 0.8）

**工费路径**（D_loh）：
- 所有节点：D = 1.0（人工制费全部归完工产品，不给在产品）
- 即 D_loh = I（单位矩阵）

### 2.5 外部投入矩阵 F

| 列 | 含义 | 数据来源 |
|----|------|---------|
| 第0列（料） | 期初金额 + 采购金额 | 物料节点 |
| 第1列（工） | 直接人工 | 工单节点 |
| 第2列（费） | 制造费用 | 工单节点 |

### 2.6 双路径求解

由于材料和工费的 D 矩阵不同，分两条路径求解：

- **材料路径**：`X_mat = (I - W × D_mat)⁻¹ × F_mat`
- **工费路径**：`X_loh = (I - W)⁻¹ × F_loh`（因为 D_loh = I）

最终合并：`X = [X_mat | X_loh]`

### 2.7 矩阵列独立性（自带成本还原特性）

由于 F 矩阵的三列（料、工、费）在求解过程中完全独立，成本永远不会"串味"——料的成本始终是料，工的成本始终是工，费的成本始终是费。

泰勒展开（Neumann Series）揭示了成本传递的层级：
$$X = F + W·F + W²·F + W³·F + ...$$

- `I × F`：本工序直接投入
- `W × F`：上一阶（跨1步）传来的成本
- `W² × F`：上两阶（跨2步）传来的成本

### 2.8 在产品成本（WIP）

$$WIP = W × (1 - D) × X_{mat}$$

只在材料维度计算 WIP（工费不产生 WIP）。

### 3.2 logic.py — CostCalculator 类

核心类，包含以下方法：

| 方法 | 功能 | 行数 |
|------|------|------|
| `__init__` | 初始化内部状态变量 | 13-31 |
| `load_data` | 读取Excel、字段映射、清洗、分组聚合 | 34-124 |

| `get_performance` | 返回性能日志 | 1653-1654 |

**calculate 方法的内部步骤**：

```
Step 1: 构建节点（物料 + 工单）
Step 2: 计算数量（期初、采购、完工、在产）
Step 3: 构建 W 矩阵（稀疏 CSR 格式）
  ├── 工单→产品：产出比例
  └── 物料→工单：消耗比例（领用 / 可供发出）
  ※ 三道保险：
    1. 列和校验（防超领）
    2. DAG 无环校验（防自循环）
    3. D 矩阵范围校验
Step 4: 构建 D 矩阵（对角阵，完工率）
Step 5: 构建 F 矩阵（期初+采购→料列，人工+制费→工费列）
Step 6: 矩阵求解（spsolve，带 fallback 到稠密）
Step 7: WIP 计算
Step 8: 结果整理
  ├── 8.1 收发存汇总表（物料维度）
  ├── 8.2 工单投入产出汇总表（工单+产品）
  ├── 8.2b 工单-产品-材料三维明细（新增）
  ├── 8.3 成本明细（节点维度）
  ├── [可选] 逐步结转法
  ├── [可选] 超级成本还原
  └── [可选] 销售成本计算
```

## 四、关键设计决策与修正记录

### 4.1 可供发出数量的口径

$$可供发出数量 = 期初 + 采购 + 完工（W矩阵用投入产出明细的完工数量）$$

这是 W 矩阵消耗比例的分母。期中把"在产品数量"包含在分母中的原因是为了确保价值流经在产品。

### 4.2 完工数量的两个口径

| 口径 | 来源 | 用途 |
|------|------|------|
| `finished_qty_for_w` | 投入产出明细 | W 矩阵计算 |
| `finished_qty_for_report` | 产成品入库明细（优先）| 收发存报表 |

### 4.3 工费全额归完工产品

人工和制造费用不产生 WIP，全部归入完工产品成本。这通过 D_loh = I 实现。

### 4.4 产出比例的计算方式

代码中工单→产品的产出比例 = 该产品完工数量 / 工单总完工数量（按完工数量比例分配），而不是理论文档中简单的 100%。这是对多产品工单场景的修正。

### 4.5 三道安全保险

```python
# 保险1：列和校验（防超领导致成本发散）
# 保险2：DAG无环校验（防自循环导致矩阵不可逆）
# 保险3：D矩阵范围校验（完工率必须在[0,1]之间）
```

### 4.6 销售成本的可供发出口径

销售成本使用 `sales_available_qty`（报表口径：期初+采购+产品完工数量），与 W 矩阵的 `available_qty`（期初+采购+材料完工数量）区分开来。
