# 字段速查——用来帮用户做映射

当你跑完 `--dry-run` 发现某张表有 `missing` 字段时，翻到这里查：
- 这个字段是干嘛的
- 用户 Excel 里可能叫什么（别名）
- 你要怎么填 `mapping_overrides`

## 先记住这两点

1. **类型不用较真**。表里写的 `int`/`float`/`str` 是程序内部清洗后的目标类型，ETL 会自动转（`pd.to_numeric`、`.astype(str)`）。用户 Excel 里年份是 `"2025"` 也能正常跑。

2. **可选字段缺了不碍事**。比如 `销售金额` 没填，程序用 0 继续跑，但毛利率分析会显示 N/A。你在 Phase 1 应该提前告诉用户："这个字段没有的话，毛利率算不出来哦。"

## 6 张表都得有，少一张都不行

`mapping_overrides` 的写法：`{"表名": {"标准字段名": "用户Excel里对应的列名"}}`

## 1. 采购入库明细 (`purchase`)

| 标准名 | 目标类型 | 是干嘛的 | 用户可能叫它... |
|--------|---------|---------|---------------|
| 年度 | int | 哪年的数据 | 年, year |
| 月份 | int | 哪月的数据 (1-12) | 月, 期间, month |
| 存货编码 | str | 物料编号 | 物料编码, 料号, 品号, 物料代码, 存货代码, 编码 |
| 采购数量 | float | 本期采购入库了多少 | 采购.*数量, 入库.*数量, 实收数量 |
| 采购金额 | float | 本期采购入库金额 | 采购.*金额, 实收金额 |

## 现成的 mapping_overrides 例子，照着写

```json
{
  "mapping_overrides": {
    "purchase": {
      "采购数量": "实收数量",
      "采购金额": "金额"
    },
    "io": {
      "工单号": "加工单号",
      "材料编码": "原材料编码"
    },
    "sales": {
      "存货编码": "物料编码"
    }
  }
}
```
