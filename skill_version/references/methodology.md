# 成本核算方法论

> 供 AI 深入理解矩阵成本核算引擎的数学原理。详细推导见 `doc/theory.md`。

## 1. 核心思想：物料与工单的二象性

传统成本模型认为"物料被物料消耗"，但 3.0 模型反过来：

```
原材料A → MO1(工单) → 半成品B → MO2(工单) → 成品C
```

- 物料不是被物料消耗的，**物料是被工单消耗的**
- 物料也不是从物料产出的，**物料是从工单产出的**

因此 W 矩阵同时包含物料和工单，构成统一的二部图网络。

## 2. W 矩阵的双区结构

```
           物料列        工单列
物料行  [   0      ]  [ 产出比例  ]
工单行  [ 消耗比例  ]  [    0     ]
```

- **右上角**（物料行×工单列）：工单→产品 产出比例
- **左下角**（工单行×物料列）：物料→工单 消耗比例（领用/可供发出）

## 3. D 阀门矩阵

对角矩阵，控制完工/在产的成本分配策略：

- **材料**：D = 完工 / (完工 + 在产) — 材料按完工率分配
- **人工/制费**：D = 1.0 — 工费全额归完工，不给在产

## 4. F 外部投入矩阵

N×3 矩阵（N = 物料数 + 工单数）：

| 列 | 来源 |
|----|------|
| 第0列 材料 | 期初材料 + 采购金额 |
| 第1列 人工 | 期初人工 + 工单直接人工 |
| 第2列 制费 | 期初制费 + 工单制造费用 |

## 5. 双路径求解

由于材料/工费的 D 矩阵不同：

- **材料路径**：`X_mat = (I - W × D_mat)⁻¹ × F_mat`
- **工费路径**：`X_loh = (I - W)⁻¹ × F_loh`（因为 D_loh = I）

最终合并：`X = [X_mat | X_loh]`

## 6. 列独立性（自带成本还原）

F 的三列在求解过程中完全独立，成本永远不会"串味"：
- 料的成本始终是料
- 工的成本始终是工
- 费的成本始终是费

泰勒展开揭示了成本传递的层级：
```
X = F + W·F + W²·F + W³·F + ...
```
- `I × F`：本工序直接投入
- `W × F`：上一阶传来的成本
- `W² × F`：上两阶传来的成本

## 7. 三大安全校验

| 校验 | 检查什么 | 为什么 |
|------|---------|--------|
| 列和 ≤1 | W 每列之和不超过 1 | 防超领导致成本发散 |
| 对角线=0 | W 不存在自环 | 防自循环导致矩阵不可逆 |
| D∈[0,1] | 完工率在有效范围 | 防非法完工率扭曲成本 |

## 8. 逐步结转法

平行结转法的补充：`S = (I + W×D) × F`。

- S_人工 = (I + W×D) × F_工
- S_制费 = (I + W×D) × F_费
- S_材料 = X总 - S_人工 - S_制费

## 9. 超级成本还原

将每个原始成本来源作为独立维度构造 F 矩阵：
- 每个物料的期初 → 独立维度
- 每笔采购 → 独立维度
- 每个工单的人工/制费 → 独立维度

对每个维度独立求解 `X_i = (I-WD)⁻¹ × F_i`，精确追踪每个原始成本去了哪些产品。
