# 故障排查

## 字段相关问题

### "缺少必要字段: ['X']"
自动字段匹配未能找到某表的必要字段。

**原因**：
- Excel 列名与标准名差异太大
- 列名包含特殊字符或换行符

**解决**：
在 config.json 中添加 `mapping_overrides`：
```json
{
  "mapping_overrides": {
    "io": {
      "工单号": "生产订单编号",
      "产品编码": "产出物料代码"
    }
  }
}
```

### 自动匹配到的字段不对
smart_match 使用正则权重匹配，可能误匹配。

**解决**：
同样是使用 `mapping_overrides` 覆盖错误的匹配。

## 矩阵校验问题

### "W矩阵列和超过1" — 超领警告
表示某个物料被多个工单领用，领用比例之和超过 100%。

**影响**：成本可能在流转中被放大。

**排查**：
1. 检查投入产出明细中该物料的领用数量是否正确
2. 检查可供发出数量（期初+采购+完工）是否偏小
3. 是否漏记了期初或采购数据

### "W矩阵存在自环" — 自循环错误
表示工单领用了自己产出的物料。

**排查**：
1. 检查 BOM 数据：材料编码是否误填为产品编码
2. 检查工单号与产品编码是否混淆

### "D矩阵存在非法值" — 完工率异常
完工率 = 完工数量 / (完工数量 + 在产数量)，必须在 [0, 1] 之间。

**排查**：
1. 检查完工数量是否有负值
2. 检查在产数量是否有负值
3. 完工+在产=0 时默认为 1.0，但不应出现此情况

## 结果问题

### "未找到任何月份数据"
所有上传的表中，年度/月份字段都是 0 或空。

**排查**：
1. 检查 Excel 中是否有年度和月份列
2. 检查字段映射是否正确（年度→年份列，月份→月份列）
3. 年度/月份是否被映射到了非数字列

### 收发存中期末金额为 0
可能原因：
1. 采购数量和完工数量都是 0 → 没有收入
2. W 矩阵消耗比例为 0 → 没有领用关系
3. F 矩阵全是 0 → 没有外部投入

### 矩阵求解失败 (LinAlgError)
回退到稠密矩阵求解仍失败。

**排查**：
1. 检查是否存在循环依赖（自环）
2. 检查是否所有节点都没有外部投入（F=0）
3. 节点数是否过大导致内存不足

## 性能问题

### 计算耗时过长
对于万级节点，稀疏求解通常很快。如果慢：

1. 检查是否安装了 SciPy（spsolve 比稠密求解快很多）
2. 检查是否有大量全零行/列
3. 逐步结转法和超级成本还原会增加额外计算量

### 内存不足
矩阵以 CSR 格式存储，内存占用 ≈ 非零元数 × 16 字节。

- 1000 物料 + 500 工单：约 24KB（极小的矩阵）
- 10000 物料 + 5000 工单：约 2.4MB
- 只有在节点数达到数十万级别时才需关注内存
